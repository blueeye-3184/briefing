# 📅 Daily Gemini-Notion Briefing Automation (Grand Final Version)

이 프로젝트는 **Google Gemini AI**와 **Notion API**를 결합하여, 매일 정해진 주제에 대해 전문가 수준의 분석 리포트를 생성하고 노션 페이지에 자동으로 기록하는 시스템입니다. **DDD/TDD 아키텍처**와 **Strict Type Safety**를 적용하여 산업용 수준의 안정성을 확보했습니다.

## 🚀 주요 기능
- **매일 자동 실행**: GitHub Actions를 통해 매일 아침 **08:17 (KST)**에 실행됩니다 (트래픽 분산 최적화).
- **학술적 심층 분석**: 단순히 정보를 요약하는 수준을 넘어, **공백 포함 5,000자 내외**의 대용량 학술 리포트를 생성합니다.
- **실시간 학술 DB 연동**: **Google Search Grounding** 기능을 통해 대한건축학회(AIK), DBpia, RISS 등 전문 학술 사이트의 데이터를 실시간으로 참조합니다. (`v1beta` 엔드포인트 및 도구 폴백 로직 적용)
- **5중 에러 방어 체계**: 2026년 표준 모델(**Gemini 2.5 Flash**)을 주력으로 하며, 5단계 모델 폴백, 랜덤 지터, 지수 백오프 및 **도구 기반 자동 재시도**를 통해 API 이슈를 완벽히 회피합니다.
- **노션 자동 구조화**: 기존 페이지 검색 기능을 통해 중복 없이 '년-월 > 주차 > 요일' 계층 구조를 유지하며, 대용량 텍스트를 블록 단위로 자동 분할하여 저장합니다.


## 👥 7인 위원회 및 역할자 정의 (Stakeholders)
본 시스템은 다음 7인의 관점을 반영하여 설계되었으며, **2026-04-17 최종 세대 교체 승인**을 획득했습니다.
1.  **CEO (경영자)**: 비즈니스 가치 및 ROI 최적화
2.  **CTO (기술책임자)**: 타입 무결성 및 시스템 확장성 확보
3.  **일반 독자**: 가독성 및 전문 용어의 쉬운 풀이
4.  **개발자**: 유지보수 용이성 및 코드 품질 (Zero Technical Debt)
5.  **코드 안정성 감시자**: API 명세 준수 및 보안 철저 (.env 보호)
6.  **사용자 (User)**: 실무 활용성 및 UX 편의성
7.  **박사과정 (연구자)**: 학술적 엄밀성 및 데이터 재현성

## 📂 파일 구조 (Modular Architecture)
모든 자산은 프로젝트 루트에 최적화되어 관리됩니다.
- `briefing_auto.py`: 제미나이 분석 및 노션 연동 핵심 로직 (Type Safe)
- `pyrightconfig.json`: Pylance Strict Mode 설정을 위한 정적 분석 가이드
- `tests/`: 시스템 무결성 검증을 위한 DDD/TDD 테스트 수트
- `.env`: 로컬 개발용 환경 변수 관리 (Git 제외 대상)
- `update.md`: 프로젝트 업데이트 및 릴리즈 노트 관리
- `LOGLIST.md`: 날짜별 브리핑 실행 이력 및 API 응답 로그 아카이브

## 📜 로그 및 데이터 관리
본 시스템은 모든 실행 결과와 API 통신 로그를 `log list/` 폴더에 체계적으로 기록합니다.
- **실시간 이력 확인**: 상세한 브리핑 이력 및 개발 로그는 [LOGLIST.md](./LOGLIST.md)에서 확인할 수 있습니다.
- **데이터 보존**: 생성된 리포트 원문(`.txt`)과 참조 자료(`.pdf`)는 날짜별 폴더에 영구 보관됩니다.

## 🛠 실행 및 테스트 (Getting Started)

### 1. 테스트 실행 (무결성 검증)
```bash
# 프로젝트 루트에서 실행
pytest tests/
```

### 2. 로컬 실행
1. `.env.example` 파일을 복사하여 `.env` 파일을 생성하고 API 키와 토큰을 입력합니다.
2. 아래 명령어를 실행합니다.
```bash
python briefing_auto.py
```
