## [Decision ID: T1-20260622-01] Slack 연동 누출 보완 및 예외 전파 체계 도입
- **일시**: 2026-06-22
- **상태**: **비준 완료 (Approved)**
- **결정 사항**:
    1. GitHub Actions 워크플로우에 `SLACK_WEBHOOK_URL` 환경 변수 주입 추가.
    2. `briefing_auto.py` 스크립트 내 예외를 명시적으로 상위 전파(`raise e`)하여 자동 실행 실패 시 Actions에서 정상 감지 가능하도록 처리.
    3. 2026년 6월 지원 모델 목록(`gemini-pro-latest` 추가, `gemini-2.5-flash-lite` 제거) 및 Notion 자식 블록 조회 로직 도입에 부합하도록 테스트 코드 전수 수정.
    4. 노션 구조 내 `2026년 06월` 중복 생성 원인 파악 및 최신 데이터 누락 없음을 확인.
- **위원 의견**:
    - **Lead Architect**: "동일 이름의 폴더가 다른 부모 아래 생성되는 문제를 '부모 ID 대조 검색(GET /blocks/{id}/children)' 도입으로 영구 해결했으며, 과거 잘못 중첩 생성된 폴더의 정리만 수동으로 수행하면 됨."
    - **Quality Director**: "그동안 Actions 상에서 실패 시에도 성공으로 보이던 Silent Success 결함을 `raise e` 추가로 종식시킴. 유효성 검사 성공 및 테스트 전수(9개) 통과 비준 완료."
    - **Integration Dev**: "GitHub Actions와 Slack 실시간 알림 간의 누락된 데이터 흐름(SLACK_WEBHOOK_URL) 주입 완료."
